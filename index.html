What is Test Driven Development?

Test Driven Development (or TDD) is the practice of software development guided by tests.

What is a test?

Code written to exercise existing code and ensure it performs the expected behavior.

Example Test


            var thingReturnsTrue = function() {
              return true;
            };

            // Jasmine
            describe('Thing returns true', function() {
              it('is true', function() {
                expect( thingReturnsTrue() ).toBe(true);
              });
            });

            // QUnit
            test('Thing returns true is true', function() {
              equal(thingReturnsTrue(), true, 'It should equal true.');
            });

Why Use Test Driven Development?

  • Ensures coverage of any code written
  • Enables code confidence
  • Encourage cleaner code and architecture
  • Projects yield fewer bugs as they grow in complexity

Code Confidence

  • You should be able to deploy your master branch/trunk at any time.
  • A well designed test suite with a high degree of coverage eliminates, or at least strongly mitigates the fear of change.
